# 基本介绍

文档修订用于在编辑过程中记录“新增 / 删除”的变更痕迹，并提供审阅与定稿能力：您可以查看每一次修改来自谁、发生在何处，并对单条或全部修订执行“接受 / 拒绝”。

> 说明：修订（revision）与历史版本（versions）是两套不同能力：修订是正文内部的变更标记；版本是协作场景下的时间点快照与回溯恢复。见：[文档历史版本](./versions)。

## 功能清单

- 记录插入/删除变更（文本修订 + 节点修订）
- 审阅面板展示修订列表（支持定位、导航、接受/拒绝）
- 四种标记视图（final-markup / final / original-markup / original）
- 按修订人过滤（支持 userId 或 userName）
- 对外方法：获取修订列表、设置标记视图、设置过滤人、接受/拒绝等

## 依赖与前置条件

- 修订功能不依赖协作，可在单机/离线场景使用。
- 如果在协作模式中开启了修订功能：修订标记作为正文内容的一部分会同步给协作者；为了让“修订人”信息准确，建议为每个客户端配置正确的 `options.user`。
