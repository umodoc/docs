# Default Configuration

Compared with Umo Editor and Umo Editor Next, the mobile configuration is more focused on touch-first reading, switching between read and edit modes, the save state machine, and lightweight integration. It does not include all desktop appearance and toolbar configuration items.

## Configuration Overview

- `editorKey`: editor instance identifier, used to distinguish local persisted state.
- `locale` / `theme` / `debug`: global display, theme, and debugging capability configuration. See [Localization Configuration](./locale), [Theme Configuration](./theme), and [Debug Configuration](./debug).
- `dicts`: dictionary configuration. See [Dictionary Configuration](./dicts).
- `page`: page configuration. See [Page Configuration](./page).
- `document`: document configuration. See [Document Configuration](./document).
- `echarts`: ECharts configuration. See [ECharts Configuration](./echarts).
- `templates`: template configuration. See [Document Template Configuration](./templates).
- `cdnUrl` / `shareUrl`: resource base URL and share URL configuration.
- `file`: file upload and preview configuration. See [File Upload Configuration](./file).
- `user` / `users`: user information configuration. See [User Configuration](./user) and [Related Users Configuration](./users).
- `extensions` / `disableExtensions`: extension add/remove configuration. See [Feature and Extension Configuration](./extensions).
- `translations`: localization override configuration. See [Localization Configuration](./locale).
- `onLeave` / `onSave` / `onRename` / `onFileUpload` / `onFileDelete`: callback method configuration. See [Method Configuration](./methods).

## Key Differences from Desktop

- `debug` <Badge theme="primary">Mobile-Only</Badge>: controls whether the built-in DevTools entry is available in the "More" panel.
- `onLeave` <Badge theme="primary">Mobile-Only</Badge>: handles mobile back navigation, page leaving, and navigation behavior after unsaved-change confirmation.
- `onRename` <Badge theme="primary">Mobile-Only</Badge>: handles document rename from the "More" panel.
- `page.layouts` defaults to `['web', 'page']` <Badge theme="warning">Different on Mobile</Badge>, which means mobile enters web layout by default; desktop usually prioritizes paged view.
- `file.preview` <Badge theme="warning">Different on Mobile</Badge>: mobile does not support third-party preview through URL templates.

## Configuration Items Not Supported on Mobile

- `skin` <Badge theme="danger">Not Supported</Badge>
- `height` <Badge theme="danger">Not Supported</Badge>
