## Configuration Description

### dicts.fonts

**Description**: Font list, used to provide selectable fonts for users.

**Type**: `Array`

**Configuration**:

- `item.label`: `String` or `Object`, the display name of the font.
- `item.value`: A valid [CSS font-family](https://developer.mozilla.org/en-US/docs/Web/CSS/font-family) value.
- `item.url`: `String`, optional. Download URL for the font file (used to download and enable the font in the browser). <Badge theme="success">Added in v10.1.0</Badge>
- `item.format`: `String`, optional. Font format for `item.url`, common values: `woff2`, `truetype`, `opentype`. <Badge theme="success">Added in v10.1.0</Badge>

**Font Download Notes**:

- When a font is not installed locally and `item.url` is configured, users can trigger download and enable it in the current browser.
- Downloaded fonts are recorded in local storage and reused on subsequent visits to avoid repeated downloads.
- Downloaded fonts are only available in the current browser context. Due to browser security limits, they cannot be automatically installed into the operating system font library.
- If you need the same font in other browsers or in exported files, provide an external download link and let users install it manually at the OS level.

**Recommended Practices**:

- Use publicly distributable font URLs with CORS support. In intranet environments, self-host the font files to ensure accessibility.
- Prefer `woff2` for smaller size; use `ttf/otf` when `woff2` is not available.
- For commercial fonts, ensure you have proper licensing before distribution to avoid legal risks.

<Callout type="warning" emoji="⚠️">
  You are responsible for validating font licensing and URL availability. Broken URLs, CORS restrictions, or slow responses will affect download and enablement experience.
</Callout>

**Configuration Example**:

```js
{
  dicts: {
    fonts: [
      {
        label: { en_US: 'Noto Serif SC', zh_CN: '思源宋体' },
        value: 'Noto Serif SC',
        url: 'https://fonts.gstatic.com/s/notoserifsc/v35/H4cyBXePl9DZ0Xe7gG9cyOj7uK2-n-D2rd4FY7SCqyWv.ttf',
        format: 'truetype',
      },
      {
        label: { en_US: 'LXGW WenKai', zh_CN: '霞鹜文楷' },
        value: 'LXGW WenKai',
        url: 'https://raw.githubusercontent.com/lxgw/LxgwWenKai/master/fonts/TTF/LXGWWenKai-Regular.ttf',
        format: 'truetype',
      },
    ],
  },
}
```

### dicts.colors

**Description**: Color list, used to provide selectable colors for users, including font colors and background colors.

**Type**: `Array`

**Configuration**: Valid [CSS color](https://developer.mozilla.org/en-US/docs/Web/CSS/color_value) values.

### dicts.lineHeights

**Description**: Line height list, used to provide selectable line heights for users.

**Type**: `Array`

**Configuration**:

- `item.label`: `String` or `Object`, the display name of the line height.
- `item.value`: A valid [CSS line-height](https://developer.mozilla.org/en-US/docs/Web/CSS/line-height) value.

### dicts.symbols

**Description**: Special character list, used to provide selectable special characters for users. Before v2.0.0, it was `dicts.specialCharacters`.

**Type**: `Array`

**Configuration**:

- `item.label`: `String` or `Object`, the group name of the special characters.
- `item.value`: All special characters under this group.

### dicts.emojis

  **Note**: The default emoji list was adjusted in v6.0.0.
</Callout>

**Description**: [Emoji](https://en.wikipedia.org/wiki/Emoji) list, used to provide selectable emojis for users.

**Type**: `Array`

**Configuration**:

- `item.label`: `String` or `Object`, the group name of the emojis.
- `item.value`: All emojis under this group, separated by spaces.

**Database Support**: Emojis are essentially Unicode characters with a length of 4 bytes, requiring database support for storage. [How to get emojis in MySQL with utf8mb4]https://juejin.cn/post/7106906047670910989). You can also disable emoji insertion through the menu with the following configuration, but this does not prevent users from inserting emojis via copy-paste or input methods.

<Tabs items={['Global Configuration', 'SFC Configuration']}>
<Tabs.Tab>
```js
{
  toolbar:{
    disableMenuItems: ['emoji']
  }
}
```
</Tabs.Tab>
<Tabs.Tab>
```vue
<template>
  <umo-editor 
    :toolbar="{
      disableMenuItems: ['emoji']
    }" 
  />
</template>
 
<script setup>
import { UmoEditor } from '@umoteam/editor'
</script>
```
</Tabs.Tab>
</Tabs>

### dicts.pageSizes

**Description**: Page size list, used to provide selectable page sizes and default page size.

**Type**: `Array`

**Configuration**:

- `item.label`: `String` or `Object`, the name of the page size.
- `item.width`: Page width in centimeters (cm).
- `item.height`: Page height in centimeters (cm).
- `item.default`: Whether it is the default page size.
